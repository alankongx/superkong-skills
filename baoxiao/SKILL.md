# 出差报销处理技能

## 适用场景

用户提交出差费用报销，需要逐一收集发票/凭证、核对金额、填写标准报销明细表。

---

## 一、报销流程

### 第一步：启动确认

收到报销请求后，先询问：
1. 出差人姓名（用于文件命名）
2. 出差日期区间（格式：YYYYMMDD-MMDD）
3. 是否有现成的报销明细表模板？路径是？

### 第二步：逐项收集费用

引导用户逐类提交发票/凭证。每收到一张，立即提取并确认：
- 费用日期
- 费用类型（见下方分类）
- 金额（人民币，**含税价**）
- 发票号（如有）
- 凭证类型（增值税发票 / 电子普通发票 / 信用卡账单 / 截图）

收集完一项后，简短确认，然后继续问「还有其他费用吗？」

### 第三步：缺项盘点

用户说「暂无其他」时，主动列出：
- ✅ 已有发票的项目
- ⚠️ 缺发票但有信用卡凭证的项目
- ❓ 尚未确认的项目（如越南住宿）

### 第四步：写入 Excel

所有费用确认完毕后，用 Python（`python3.11`）操作 openpyxl 写入报销明细表。

**输出文件命名规则：**
```
{姓名拼音}_{出差起始日期YYYYMMDD}-{结束日期MMDD}_报销明细表.xlsx
示例：孔德化_20260417-0514_报销明细表.xlsx
```

**输出路径：** `~/Downloads/`

### 第五步：补充发票号

机票发票有时会在会话中途才补齐（用户上传 PDF 后）。收到后立即更新对应行的发票号字段，不需要重新生成整个表格。

## 二、费用类型与报销规则

| 费用类型 | 是否报销 | 凭证要求 | 备注 |
|---------|---------|---------|------|
| 机票（国际/国内） | ✅ | 电子普通发票 | 用**含税价**；多张发票（机票+保险）发票号用 `/` 分隔 |
| 签证费 | ✅ | 发票 | — |
| 国内打车（滴滴等） | ✅ | 电子发票 | — |
| 越南打车（Grab） | ✅ | 信用卡账单截图 | Grab 无法开发票，信用卡凭证即可 |
| 国内住宿（如昆明中转） | ✅ | 发票 | 线上退款+线下支付的情况：以实际支付为准，备注说明 |
| 越南住宿 | ❌ 不报销 | — | 公司统一安排，不纳入个人报销 |
| 业务招待费（请供应商吃饭等） | ✅ | 信用卡账单截图 | 备注注明事由和对象 |
| 机场礼品 | ✅ | 发票 | 备注用途 |

## 三、常见澄清场景

**含税 vs 不含税：**
- 发票上金额与用户报的金额不符时，先提醒差异，询问「报销金额用含税价还是不含税？」
- 默认用**含税价**

**线上退款 + 线下支付：**
- 用户说「线上退款了，我是线下付的」→ 按线下支付金额报销，备注「线下支付，线上退款」

**越南 VND 转人民币：**
- 用信用卡账单上的人民币记账金额，不自行换算汇率

**发票分批到：**
- 机票发票（尤其5月13-14日回程）可能在会话后期才补齐
- 先记录金额，发票号留空，补齐后再更新 Excel 对应行

**多张保险发票：**
- 一次出行可能有 2 张保险发票，都要记录，发票号用 `/` 分隔写在同一格

---

## 四、Excel 填写规范

- 每笔费用独占一行
- 多张发票对应同一笔费用时：发票号用 `/` 分隔，写在同一行
- 金额列：人民币，保留两位小数
- 汇总行：通常在最后一行（如第25行），用 SUM 公式，不手动填
- 凭证数量：有几张凭证就填几，无发票但有信用卡截图也计入

---

## 五、注意事项

- 处理 PDF 发票时，用 `pdf` 工具读取，不用 `read`（二进制）
- 处理图片发票时，用 `image` 工具读取
- Excel 操作使用 `python3.11` + `openpyxl`，不用 `python3`（系统链接损坏）
- 中间处理文件（PDF 副本等）放 `~/.easyclaw/workspace-baoxiaozhuli/`
- 最终输出文件放 `~/Downloads/`
