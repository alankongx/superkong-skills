## ⚠️ 强制规则(不得违反)

### 1. 零件不得省略
- **客户 RFQ 里的所有零件必须全部输出,一个不能少。**
- 不得以任何理由自行过滤零件(不论 EAU 高低、Phase 优先级、是否有图纸、是否是变体)。
- 如客户 RFQ 有 437 个零件,询价单必须有 437 行(英文区)+ 437 行(中文区)。
- **唯一例外**:用户明确指定只处理部分零件时,才可按指示过滤。

### 2. 输出路径:macOS 下载文件夹
- 所有生成的 xlsx 文件默认保存到 `~/Downloads/`(即 `/Users/<用户名>/Downloads/`)。
- 不得保存到工作区目录或其他路径,除非用户明确指定。
- 生成后必须通过 `MEDIA:` 发送文件给用户。

### 4. 英文区与中文区严格隔离(v2.8 新增强制规则)
- **英文区列表头**:一律纯英文,禁止出现任何中文字符(包括冻结行)
- **中文区列表头**:一律纯中文,禁止出现任何英文字段名
- **信息行隔离**:`Opp. No / Date / Customer / Country / Annual Qty` 这一行出现在英文区上方为纯英文;出现在中文区上方为纯中文(商机编号/日期/客户/来源国/年需求量)
- **单位字段**:英文区单位保留英文(pc/set/pcs);中文区单位必须转换为中文(个/套/张/批...),禁止保留英文
- **Notes / 备注**:Notes 跟在英文明细尾部,内容纯英文;备注跟在中文明细尾部,内容纯中文;两者不得合并或混排
- **价格列币种**:英文区单价/模具费列标题括号内写币种英文(如 `Unit Price (USD)`);中文区对应列括号内必须写**人民币**(如 `单价(人民币)`、`模具费(人民币)`),禁止在中文区写 `(美元)` 或 `(USD)`

### 5. 输出文件命名规则(优先级从高到低)
1. **用户明确指定文件名** → 完全按用户指定输出
2. **仅中文版命名** → `YYYYMMDD_{商机编号}_询价单_{中文品名}{零件数量}_{版本号}.xlsx`
   - 例:`20260522_O5818_询价单_镁压铸零件x2_v1.xlsx`
   - **`RFQ` 改为「询价单」**（中文字）
   - **品名必须为中文**（如「镁压铸零件」「冲压件」「注塑件」），不得使用英文产品名
   - **零件数量格式**：品名后紧跟 `x{主零件数量}`，字母 x 小写，数字为本次询价的主零件总数
     - 2个主零件 → `x2`，3个主零件 → `x3`，以此类推
     - 主零件指顶层独立零件（父项），不含子件/变体
   - **版本号格式**：`_v1`（第一次生成）、`_v2`（第二次）、`_v3`……以此类推
3. **中英双语版 / 仅英文版命名** → `YYYYMMDD_{商机编号}_RFQ_{英文品名}_{版本号}.xlsx`
   - 例:`20260505_O5779_RFQ_Blow Molded Reservoir_v5.xlsx`
4. **若品名暂时无法确定** → 仅中文版用 `YYYYMMDD_{商机编号}_询价单_{版本号}.xlsx`；英文/双语版用 `YYYYMMDD_{商机编号}_RFQ_{版本号}.xlsx`
5. **兜底** → `询价单_FINAL_{日期}.xlsx`

### 6. 询盘文件夹自动整理(v2.9 新增强制规则)
每次生成询价单后,**必须**在 `~/Downloads/` 下自动创建询盘文件夹并整理文件:

**文件夹命名格式**:`YYYYMMDD_{商机编号}_{中文产品名}` 
- 例:`20260522_O5818_镁压铸零件`
- 日期为当天8位数字,产品名用简洁中文(与询价单文件名中的中文品名保持一致)

**子文件夹结构**:
```
YYYYMMDD_{商机编号}_{中文产品名}/
├── 原始文件/      ← 用户提供的所有原始资料
└── 询价资料/      ← 生成的询价单 xlsx
```

**原始文件夹**:将用户本次提供的所有文件移入,包括:
- 客户询价 ZIP 包、PDF 图纸、Excel RFQ 等原始附件
- 用户发送的截图/图片(从 inbound 媒体路径 **复制**,不可移动)
- 若文件在 inbound 媒体目录(`~/.easyclaw/media/inbound/`)则只能 cp 不能 mv,并重命名为有意义的中文文件名(如「客户询价邮件截图.jpg」)

**询价资料文件夹**:将生成的 xlsx 文件移入(mv)

**执行时机**:在发送 `MEDIA:` 给用户之前完成文件夹创建与文件移动,然后再发送 xlsx。

### PDF 处理流程(零 token,完全本地)
```
用户提供 PDF
    │
    ▼
pdfminer 提取文本
    │
    ├─ 文本量 ≥ 30字符 → [矢量型] 直接使用提取结果
    │
    └─ 文本量 < 30字符 → [图片扫描型] 自动触发 OCR 流程
            │
            ├─ macOS: qlmanage 渲染为 PNG(系统内置,无需安装)
            ├─ Linux: pdftoppm 渲染为 PNG
            └─ Tesseract OCR 识别文字 → 返回文本
```

**关键设计原则:**
- ❌ 不调用任何云端 AI 来读图纸(节省 token)
- ✅ pdfminer → 矢量 PDF 直接提取(最快、最准)
- ✅ qlmanage + Tesseract → 扫描版回退(macOS 内置渲染,完全免费)
- ✅ 在线翻译(Google Translate)保留,准确度第一

### 翻译策略(准确度优先)
| 优先级 | 方式 | 说明 |
|--------|------|------|
| 1 | 本地工程术语词典 | 命中则直接使用,最快最准 |
| 2 | Google Translate 在线兜底 | 未命中则调用,`USE_ONLINE_TRANSLATE=True`(默认) |
| 注意 | 不得关闭在线翻译 | 准确度第一,省钱不是第一位的 |

## 功能说明
针对机械加工行业采购场景,自动化完成从图纸到询价单的全流程:
1. 自动识别上传的工程图纸,提取零件编号、名称、材质、数量、公差、表面处理等参数
2. 自动识别客户原始询价单,提取国别、客户背景、特殊要求、交付时间等商务信息
3. **智能补列**:客户原始信息中有、但标准模板里没有的字段(如零件重量、交期、认证要求等),自动作为额外列追加到表格末尾
4. 生成格式:第一列 Sr. No(层级序号)→ 英文零件信息N行 → 英文备注 → 中文零件信息N行 → 中文注释
5. 输出 **xlsx** Excel 文件(非 csv),可直接用 Excel / 飞书表格打开
6. **按 Phase 颜色区分优先级**:A=蓝(最高量),B=绿,C=黄,D=橙,E-I依次降低,方便供应商识别重点
7. **层级结构完整呈现**:首列输出 Sr. No(与客户 BOM 一致的层级编号 1、1.1、1.2、2、2.1.1 等),描述列按层级深度左侧缩进,供应商一眼看清父子关系
8. **默认中英双语术语规范化**:中文版必须将原材料、表面处理、主要工艺翻译成中文
9. **默认最终校对**:输出前自动执行一轮采购/工程口径校对

## 使用流程
1. 用户发送指令:`生成询价单`
2. 识别客户原始询价单文件名(用于输出命名)
3. 询问商机编号(用于生成内部编号)
4. **必须询问语言版本**:每次生成询价单前,必须先问用户选择输出语言:
   > "请问需要哪种语言版本?
   > 1️⃣ 仅中文
   > 2️⃣ 中英双语
   > 3️⃣ 仅英文"
   - 用户回复后,按对应选择生成:"仅中文"只生成中文区;"中英双语"生成英文区+中文区;"仅英文"只生成英文区
   - **不得假设默认语言,每次都必须问**
5. **自动判断来源国**(调用 `detect_country_from_files()`,不依赖用户输入):
   - 读取客户 PDF 图纸版权声明 → 提取公司所属国(最高置信度)
   - 读取客户 RFQ Excel 中的联系地址
   - 分析文件名/邮件域名中的国家关键词
   - 无法判断时才询问用户，不得擅自填写
6. 读取所有图纸文件（PDF 自动判断矢量/扫描型，扫描型自动 OCR）
7. 读取客户 RFQ 提取**全部**零件（不省略任何零件）
8. 生成初版 xlsx，执行以下默认优化：
   - 父子项关系整理
   - 中文术语翻译补全（仅当生成中文区时）
   - Phase 颜色标注
   - 标题排版规范化
9. 保存到 `~/Downloads/`，文件名按语言版本规则确定(见强制规则§5)；若用户另有指定则按用户指定
10. **自动创建询盘文件夹并整理文件**(见强制规则§6):创建 `YYYYMMDD_{商机编号}_{中文产品名}/原始文件/` 和 `询价资料/`,将原始附件移入「原始文件」,询价单 xlsx 移入「询价资料」
11. 通过 `MEDIA:` 发送文件给用户

## 来源国自动判断规则(v2.3 新增)
`detect_country_from_files(file_paths, extra_text)` 函数按以下优先级判断:

| 优先级 | 来源 | 举例 |
|--------|------|------|
| 1(高) | PDF 版权声明/公司地址 | "DOVER FUELING SOLUTIONS **UK** LTD." → 英国 |
| 1(高) | RFQ Excel 联系地址 | "Sugar Land, TX 77478" → 美国 |
| 2(中) | 文件名关键词 | `NPD0226-S-27-**HW**` 无法判断;但 `Quote_Germany.xlsx` → 德国 |
| 3(低) | extra_text 补充说明 | 用户说"这是英国客户"→ 英国 |
| 未知 | 所有来源均无线索 | **询问用户,不得自行假设** |

支持的国家关键词库涵盖:英国(UK Ltd / Dover UK / .co.uk)、德国(GmbH / .de)、法国、荷兰、美国(Inc. / LLC / Corp.)、日本、韩国、中国、越南、印度等 20+ 国家。

## Internal No. 编号规则(v2.9 更新)
- 首列标题固定为:`Internal No.`(英文区)/ `内部编号`(中文区)
- **编号结构为两级**,总成与子件各自位数按实际数量自适应,格式如下:
  - **总成(父项)**:`{商机编号}{总成序号}`
  - **子件**:`{总成编号}{子件序号}`
- **自适应位数规则**(扫描全表后确定,生成前一次性计算):
  - 总成总数 **≤ 99** → 总成序号用 **两位**,如 `O581401`、`O581402`
  - 总成总数 **≥ 100** → 总成序号用 **三位**,如 `O5814001`、`O5814002`
  - 任意总成下子件数 **≤ 99** → 该总成的子件序号用 **两位**,如 `O58140101`
  - 任意总成下子件数 **≥ 100** → 该总成的子件序号用 **三位**,如 `O5814010001`(三位子件)
  - 两项规则独立判断:总成位数和子件位数互不影响
- **设计原则**:编号永不重叠,默认两位,仅在超限时升三位
- 序号与商机编号之间**不加连字符**
- 商机编号必须由用户提供,不能自行编造
- **中文区表头**:`Internal No.` 对应列的中文标题固定为 `内部编号`,不得写 `德闻编号`

## 对外发供应商版本的脱敏规则
- 若询价单是发给供应商的,默认视为**外发版本**
- 外发版本中,不得出现以下客户识别信息:
  - 客户公司全名
  - 客户品牌名
  - 联系人姓名
  - 邮箱地址
  - 电话号码
  - 详细地址
- 应改写为概括性表述,例如:
  - `某美国水培植物公司`
  - `a U.S. hydroponics company`
  - `某美国能源设备客户`
- 若用户明确授权保留客户实名信息,才可输出未脱敏版本

## 智能补列规则
- 标准模板列(固定16列)始终存在
- 扫描客户原始询价信息,识别出模板未覆盖的字段类型,追加为额外列
- 常见额外字段:零件重量(Weight)、交期(Lead Time)、认证要求(Certification)、Phase、Rev 等

## 默认质量要求
- 中文区域产品名称必须输出**正确中文品名**,禁止在中文名缺失时统一写成"零件"
- 翻译优先级固定为:**本地词典/规则映射 > 在线翻译兜底 > 最后保留原文**
- 中文区域材料/工艺/表面处理字段必须优先翻译为中文
- 标准牌号、材料代号(如 ASTM B633-11、EN 10346、AISI 12L14)可保留英文标准号
- 英文区域保持专业英文
- 有图纸的零件:从图纸精确提取材料/厚度/表面处理
- 无图纸的零件(客户标注 drawing missing 或文件夹内无对应 PDF):
  - 整行底色标黄(WN_FILL = FFF2CC)
  - 序号列、图号列、名称列、备注列用红色加粗字体(color C00000)
  - 英文名称前加 `[WARNING: DRAWING MISSING]`,中文名称前加 `【警告:图纸缺失】`
  - 材料/工艺字段填写 RFQ 文字信息(如有),否则填"Not specified in RFQ - to be confirmed" / "RFQ未注明,待确认"
  - 备注列注明 `[WARNING] Drawing missing - info from RFQ text only` / `【警告】图纸缺失--当前信息仅来自RFQ文本`

## 依赖
- openpyxl(xlsx 输出)
- pdfminer.six(PDF 矢量文本提取)
- pytesseract + Pillow(图片/扫描 PDF OCR)
- deep-translator(在线翻译,保持开启)
- qlmanage(macOS 系统内置,无需安装,用于渲染扫描版 PDF)
- pdftoppm(Linux 可选,poppler-utils 包含)

安装:`pip install openpyxl pdfminer.six pytesseract Pillow deep-translator`

> macOS 用户无需额外安装渲染工具,系统自带 `qlmanage`。
